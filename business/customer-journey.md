# Customer Journey Map: Identity-Sheet

## Aware
**Trigger event**: User experiences spreadsheet errors due to positional referencing or hears about identity-based referencing from colleagues
**Friction points**: Information overload about spreadsheet solutions, skepticism about new approaches, time constraints to research alternatives
**User emotions**: Frustration with current errors, hope for a better solution, uncertainty about trying something new
**Opportunities to delight**: Clear marketing that directly addresses pain points, social proof from early adopters, educational content about identity-based referencing benefits
**Metric**: Website traffic from targeted search terms ("spreadsheet errors," "alternative to Excel references"), social media mentions

## Consider
**Trigger event**: User visits the product website and reads about identity-based referencing
**Friction points**: Learning curve concerns, integration with existing workflows, cost considerations, trust in new solution
**User emotions**: Curiosity about the solution, skepticism about claims, hope that this could work, anxiety about change
**Opportunities to delight**: Free trial or demo, clear ROI calculator, case studies showing reduced errors, responsive customer support
**Metric**: Trial sign-up rate, demo completion rate, time spent on educational content

## Try
**Trigger event**: User signs up for trial and creates first spreadsheet with identity-based referencing
**Friction points**: Initial setup complexity, breaking old habits of positional referencing, understanding the new paradigm, potential technical issues
**User emotions**: Excitement about trying something new, frustration with initial learning curve, satisfaction when first successful, doubt if it's worth the effort
**Opportunities to delight**: Onboarding tutorials, responsive customer support, quick wins that demonstrate value, personalized guidance
**Metric**: Trial activation rate, time to first successful use, support ticket volume, feature adoption rate

## Adopt
**Trigger event**: User decides to use the product regularly and integrates into daily workflow
**Friction points**: Changing team's workflow, convincing others to adopt, migration of existing spreadsheets, scaling usage
**User emotions**: Confidence in the solution, relief from reduced errors, pride in adopting better practices, frustration with any remaining issues
**Opportunities to delight**: Team collaboration features, migration tools, advanced tutorials, community of users
**Metric**: Monthly active users, retention rate, feature usage depth, customer satisfaction score

## Expand
**Trigger event**: User experiences success with the product and sees opportunities for broader application
**Friction points**: Scaling to more complex use cases, integration with other systems, cost of advanced features, managing multiple users
**User emotions**: Enthusiasm for the product, desire for more capabilities, concern about costs, excitement about potential
**Opportunities to delight**: Advanced feature packages, enterprise support, API for integration, user feedback incorporation
**Metric**: Upsell conversion rate, expansion to new teams/departments, advanced feature adoption, net promoter score
